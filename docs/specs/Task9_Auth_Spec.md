## Database Reference

### Table: `users` (authentication)

| Column | Type | Description |
|--------|------|-------------|
| user_ref_no | INT | Primary key, auto-increment |
| name | VARCHAR(100) | User's name |
| email | VARCHAR(100) | Unique, for login |
| password | VARCHAR(255) | Hashed password |
| role_id | INT | FK to roles table |
| address | TEXT | User address |
| area_id | INT | FK to areas table |
| status | VARCHAR(50) | active/approved |
| user_code | VARCHAR(20) | Auto-generated by trigger |
| created_at | TIMESTAMP | Registration date |

### Table: `user_profiles` (extended info)

| Column | Type | Description |
|--------|------|-------------|
| profile_id | INT | Primary key |
| user_ref_no | INT | FK to users table |
| full_name | VARCHAR(100) | Full name |
| address | VARCHAR(100) | Address |
| area_id | INT | FK to areas table |
| phone | VARCHAR(20) | Phone number |
| profile_created_at | TIMESTAMP | Profile creation date |

### Table: `roles`

| role_id | role_name |
|---------|-----------|
| 1 | Resident |
| 2 | SME |
| 3 | Council Member |
| 4 | Council Administrator |

**Role Permissions:**
- `Resident` - Can browse and vote
- `SME` - Can list products/services
- `Council Member` - Can manage areas
- `Council Administrator` - Full access

## API Endpoints

### 1. POST `/auth.php?action=signup`

**Purpose:** Register a new user (creates records in both `users` and `user_profiles` tables)

**Request Body:**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "securepass123",
  "role_id": 1,
  "area_id": 1,
  "address": "123 Main Street, Hatfield",
  "phone": "07123456789"
}
```

**Field Mapping:**

| Field | users table | user_profiles table |
|-------|-------------|---------------------|
| name | name | full_name |
| email | email | - |
| password | password (hashed) | - |
| role_id | role_id | - |
| area_id | area_id | area_id |
| address | address | address |
| phone | - | phone |

**Validation:**
- name, email, password, role_id, area_id required
- phone optional but recommended
- Email must be valid format
- Email must not already exist
- Password minimum 6 characters
- role_id must be 1-4
- area_id must exist in areas table

**Signup Flow:**
```
1. Validate input
2. Check email doesn't exist
3. Hash password
4. INSERT into `users` → get user_ref_no
5. INSERT into `user_profiles` → link with user_ref_no
6. Return success with user data
```

**Success Response (201):**
```json
{
  "success": true,
  "data": {
    "user_ref_no": 15,
    "name": "John Doe",
    "email": "john@example.com",
    "role_id": 1,
    "user_code": "RES-0003",
    "message": "Registration successful"
  }
}
```

**Error Responses:**
```json
// 400 - Missing fields
{
  "success": false,
  "error": "Missing required fields: name, email"
}

// 400 - Email exists
{
  "success": false,
  "error": "Email already registered"
}

// 400 - Invalid email
{
  "success": false,
  "error": "Invalid email format"
}
```

---

### 2. POST `/auth.php?action=login`

**Purpose:** Authenticate user and start session

**Request Body:**
```json
{
  "email": "john@example.com",
  "password": "securepass123"
}
```

**Validation:**
- Email and password required
- Email must exist in database
- Password must match hash

**Success Response (200):**
```json
{
  "success": true,
  "data": {
    "user_ref_no": 15,
    "name": "John Doe",
    "email": "john@example.com",
    "role_id": 1,
    "role_name": "Resident",
    "user_code": "RES-0003",
    "area_id": 1,
    "message": "Login successful"
  }
}
```

**Error Responses:**
```json
// 400 - Missing fields
{
  "success": false,
  "error": "Email and password required"
}

// 401 - Invalid credentials
{
  "success": false,
  "error": "Invalid email or password"
}
```

**Session Data Stored:**
```php
$_SESSION['user_ref_no'] = $user['user_ref_no'];
$_SESSION['user_name'] = $user['name'];
$_SESSION['user_email'] = $user['email'];
$_SESSION['user_role_id'] = $user['role_id'];
$_SESSION['user_role_name'] = $user['role_name'];
$_SESSION['user_code'] = $user['user_code'];
$_SESSION['logged_in'] = true;
```

---

### 3. POST `/auth.php?action=logout`

**Purpose:** End user session

**Request Body:** None required

**Success Response (200):**
```json
{
  "success": true,
  "data": {
    "message": "Logout successful"
  }
}
```

**Actions:**
1. Unset all session variables
2. Destroy session
3. Clear session cookie

---

### 4. GET `/auth.php?action=check`

**Purpose:** Check if user is logged in

**Request Body:** None

**Success Response (200) - Logged in:**
```json
{
  "success": true,
  "data": {
    "logged_in": true,
    "user": {
      "user_ref_no": 15,
      "name": "John Doe",
      "email": "john@example.com",
      "role_id": 1,
      "role_name": "Resident",
      "user_code": "RES-0003"
    }
  }
}
```

**Response (200) - Not logged in:**
```json
{
  "success": true,
  "data": {
    "logged_in": false,
    "user": null
  }
}
```

---

### 5. GET `/auth.php?action=check_role&role=Admin`

**Purpose:** Verify user has specific role

**Query Parameters:**
- `role` - Required role to check

**Success Response (200) - Has role:**
```json
{
  "success": true,
  "data": {
    "authorized": true,
    "user_role": "Admin"
  }
}
```

**Error Response (403) - Unauthorized:**
```json
{
  "success": false,
  "error": "Access denied. Required role: Admin"
}
```

**Error Response (401) - Not logged in:**
```json
{
  "success": false,
  "error": "Not authenticated"
}
```

## Security Considerations

| Risk | Mitigation |
|------|------------|
| Password exposure | Never return password_hash in responses |
| SQL Injection | Use prepared statements for all queries |
| Brute force attacks | Consider rate limiting (optional) |
| Session hijacking | Use `session_regenerate_id()` on login |
| XSS | Sanitize all user input |
